# Public Private Boundary

The public repository is an engine/template. Private memory belongs in a private vault. [Source: README.md]

## Keep Public

- Generic scripts and tests.
- Generic `AGENTS.md` schema.
- Reusable skills.
- Synthetic examples.
- Placeholder infra templates.

## Keep Private

- Real raw captures.
- Real people, project, customer, family, health, financial, or company memory.
- Real chat IDs and server IPs.
- Deploy-key paths, OAuth usernames, token locations, and secret names.
- Source manifests pointing at private files.

Do not commit private material and rely on later deletion. Git history is part of publication. [Source: docs/publication-safety.md]
