# Publication History Safety

Deleting private files in a commit does not remove those files from Git history. If a repository has ever contained private data, do not toggle that repository public. [Source: docs/publication-safety.md]

Safe publication paths are a fresh public repository from a clean tree, or a history rewrite followed by separate verification. [Source: docs/publication-safety.md]

The public-safety checklist bans real raw captures, private project/person/family/company pages, real chat IDs, server IPs, deploy-key paths, OAuth usernames, token locations, secret names, and manifests pointing at private files. [Source: docs/publication-safety.md]
